# Техническое задание (упрощённое) для учебного проекта  
**Сайт по продаже проходок и донатов для Minecraft сервера**  
(версия для двух разработчиков без опыта, срок – 1 неделя)

---

## 1. Нетехническая часть (функциональность для пользователя)

### 1.1. Главная страница
- Краткое описание проекта.
- Навигация: **Главная**, **Игровая карта**, **Услуги**, **Правила**, **Профиль**, **Темы**.
- Кнопки **Войти** / **Регистрация** (если не авторизован).
- Переключатель языка (русский / английский) через файлы `ru.json`, `en.json`.
- Переключатель темы (светлая / тёмная) с сохранением выбора.

### 1.2. Регистрация и авторизация
- **Регистрация**  
  - Логин (уникальный)  
  - Пароль (хешируется на сервере)  
  - Minecraft никнейм (уникальный)  
- **Авторизация**  
  - Вход по логину и паролю  
- После входа роль `user` (обычный) или `support` (назначается администратором в БД).

### 1.3. Профиль пользователя
- Отображается: логин, Minecraft никнейм, дата регистрации, роль.
- **Статистика по серверам** (из игровой БД):  
  - Сервер, общее время игры (минуты), последняя активность.
- **История покупок**:  
  - Дата, сервер, тип (проходка / донат), сумма.  
  - Кнопка **Скачать чек** (файл `.txt`).
- **Смена никнейма**  
  - Поле ввода, кнопка «Изменить».  
  - Проверка уникальности (никнейм не должен принадлежать другому пользователю).
- **Смена пароля**  
  - Старый пароль, новый пароль, подтверждение.

### 1.4. Игровая карта
- Статическое изображение карты сервера (или ссылка на внешнюю динамическую карту).
- Список игроков онлайн (никнеймы и время в игре) из `game_db.stats`.

### 1.5. Услуги (донаты и проходки)
- Список серверов. При выборе сервера – доступные товары с ценами.
- **Покупка (симуляция)**:  
  - Пользователь выбирает товар → нажимает «Купить».  
  - Создаётся запись в истории покупок, генерируется чек (`txt`), услуга считается «выданной» (без реальной оплаты).

### 1.6. Правила
- Общие правила сервера (всегда видны).  
- Правила для каждого сервера – выбор через выпадающий список.  
- Внутри правил – **якоря** для быстрого перехода (например, `#pvp`, `#economy`).

### 1.7. Техподдержка
- Форма обращения: тема, описание.  
- После отправки сервер отправляет сообщение в Discord через webhook (логин пользователя, ID, тема, описание).  
- Для роли `support` в Discord доступны команды `/sendhelp` и `/closedhelp` (управление обращениями).  
- На сайте – просмотр своих тикетов (упрощённо).

## 2. Техническая часть (для разработчиков)

### 2.1. Архитектура
- **Бэкенд**: Go (`net/http` или `gin`/`echo`)  
- **Фронтенд**: HTML + CSS + нативный JavaScript (без SPA-фреймворков)  
- **База данных**: MySQL (две отдельные БД: `site_db` и `game_db`)  
- **Чеки**: сохраняются как `.txt` файлы в папке `receipts/`, путь в БД

### 2.2. Базы данных

#### 2.2.1. База данных сайта – `site_db`

**Таблица `users`**  
| Поле | Тип | Ограничения |
|------|-----|--------------|
| `id` | INT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT |
| `login` | VARCHAR(50) | UNIQUE, NOT NULL |
| `password_hash` | CHAR(60) | NOT NULL (bcrypt) |
| `role` | ENUM('user','support') | NOT NULL, DEFAULT 'user' |
| `nickname` | VARCHAR(30) | UNIQUE, NOT NULL |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP |

**Таблица `purchases`**  
| Поле | Тип | Ограничения |
|------|-----|--------------|
| `id` | INT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT |
| `user_id` | INT UNSIGNED | FOREIGN KEY (`users.id`) |
| `server_id` | INT UNSIGNED | (ссылается на `game_db.servers.id`, без FK) |
| `type` | VARCHAR(50) | NOT NULL ('pass' / 'donate') |
| `amount` | DECIMAL(10,2) | NOT NULL |
| `purchase_date` | DATETIME | DEFAULT CURRENT_TIMESTAMP |
| `details` | TEXT | NULL |

**Таблица `receipts`**  
| Поле | Тип | Ограничения |
|------|-----|--------------|
| `uuid` | VARCHAR(36) | PRIMARY KEY |
| `user_id` | INT UNSIGNED | FOREIGN KEY (`users.id`) |
| `purchase_id` | INT UNSIGNED | FOREIGN KEY (`purchases.id`) |
| `file_path` | VARCHAR(255) | NOT NULL |
| `issue_date` | DATETIME | DEFAULT CURRENT_TIMESTAMP |

**Таблица `support_tickets`** (упрощённая)  
| Поле | Тип | Ограничения |
|------|-----|--------------|
| `id` | INT UNSIGNED | PRIMARY KEY AUTO_INCREMENT |
| `user_id` | INT UNSIGNED | FOREIGN KEY (`users.id`) |
| `topic` | VARCHAR(100) | NOT NULL |
| `message` | TEXT | NOT NULL |
| `status` | VARCHAR(20) | DEFAULT 'open' |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP |

#### 2.2.2. База данных игровая – `game_db`

**Таблица `servers`**  
| Поле | Тип | Ограничения |
|------|-----|--------------|
| `id` | INT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT |
| `name` | VARCHAR(100) | UNIQUE, NOT NULL |
| `description` | TEXT | NULL |
| `rules` | TEXT | NULL (HTML с якорями) |
| `category` | VARCHAR(50) | NULL |

**Таблица `stats`**  
| Поле | Тип |
|------|-----|
| `id` INT UNSIGNED PRIMARY KEY AUTO_INCREMENT | |
| `player_nickname` VARCHAR(30) NOT NULL | (ссылается на `site_db.users.nickname`) |
| `server_id` INT UNSIGNED NOT NULL | (ссылается на `servers.id`) |
| `total_play_time` INT DEFAULT 0 | (минуты) |
| `last_active` DATETIME | |

**Таблица `donations`** (выданные донаты)  
| Поле | Тип |
|------|-----|
| `id` INT UNSIGNED PRIMARY KEY AUTO_INCREMENT | |
| `player_nickname` VARCHAR(30) NOT NULL | |
| `server_id` INT UNSIGNED NOT NULL | |
| `donate_package` VARCHAR(100) NOT NULL | |
| `amount` DECIMAL(10,2) NOT NULL | |
| `granted_at` DATETIME DEFAULT CURRENT_TIMESTAMP | |

> **Примечание:** межбазовые внешние ключи не используются. Уникальность никнеймов обеспечивается логикой приложения.

### 2.3. API (бэкенд на Go)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/register` | Регистрация (логин, пароль, никнейм) – проверка уникальности |
| POST | `/api/login` | Авторизация, установка сессии / JWT |
| GET | `/api/profile` | Информация профиля, история покупок, статистика с серверов |
| PUT | `/api/profile/nickname` | Смена никнейма (проверка уникальности) |
| PUT | `/api/profile/password` | Смена пароля |
| POST | `/api/purchase` | Симуляция покупки (server_id, type, amount, details) → чек, запись |
| GET | `/api/receipt/{uuid}` | Скачать чек (.txt) |
| POST | `/api/support` | Создать обращение → отправить в Discord webhook |
| GET | `/api/servers` | Список серверов из `game_db.servers` |
| GET | `/api/stats/{nickname}` | Статистика игрока по серверам |

### 2.4. Требования к реализации
- **Аутентификация**: сессии (cookies) или JWT, пароли – bcrypt.
- **Уникальность никнейма** – при регистрации и смене в профиле.
- **Чеки**:  
  - Генерация UUID v4.  
  - Создание `receipts/<uuid>.txt` с содержимым (пользователь, товар, сумма, дата, UUID).  
  - Запись пути в `receipts`.  
  - Отдача файла при запросе.
- **Discord webhook** – URL хранить в `.env`.
- **Локализация** – `ru.json`, `en.json` в `locales/`, подстановка через JS.
- **Темы** – класс `dark-theme` на `body`, переключение через JS.
- **Деплой** – любой хостинг с Go и MySQL (или локально).

### 2.5. Что исключено / упрощено по сравнению с исходным ТЗ
- Внутренний баланс и пополнение кошелька – **полностью убраны**.
- Раздел **«Вики»** – удалён.
- Интерактивный чат на карте – заменён статической картой.
- Сложная статистика (топы, поиск) – только время игры и последний вход.
- Чеки только в `.txt` (не PDF).
- Команды `/sendhelp`, `/closedhelp` реализованы только в Discord (не на сайте).

---

## 3. Критерии готовности

1. Регистрация, авторизация, выход работают.  
2. Никнейм уникален – нельзя зарегистрироваться или сменить на занятый ник.  
3. Покупка симулируется: создаётся чек и запись в истории, услуга «выдаётся».  
4. В профиле видна история покупок, чеки скачиваются.  
5. Есть список серверов и донатов.  
6. Обращение в техподдержку отправляет сообщение в Discord (webhook).  
7. Переключение языка (ru/en) и темы (светлая/тёмная) работает.  
8. Базы данных `site_db` и `game_db` не связаны прямыми внешними ключами.  

**Срок разработки:** 7 дней, команда 2 человека (без опыта).
