# Техническое задание для учебного проекта: сайт по продаже проходок и донатов для Minecraft сервера

## Общая концепция
Разработка сайта и серверной части для управления покупками проходки и донатов:
- Регистрация и авторизация
- Хранение логина, хеша пароля, имени аккаунта
- Пополнение кошелька с условной обработкой
- Просмотр статистики по каждому серверу
- Чеки с возможностью скачивания
- История покупок
- Возможность менять никнейм (майнкрафт имя) в профиле
- Никнейм — уникальный, проверить его перед установкой
- Использование ролей (пользователь, техподдержка)

---

## Основные требования

### 1. Пользовательский интерфейс
- **Главная страница**
  - Описание проекта
  - Навигационная карта: главная, Игровая карта и чат, услуги, правила, вики, профиль, темы
  - Кнопка "Войти" и "Регистрация"
  - Переключение языка (ru/en) через файлы `ru.json`, `en.json`
  - Переключение темы (светлая/темная)
  
- **Форма регистрации**
  - Логин
  - пароль ( он хешируется в базе данных)
  - Имя аккаунта

- **Авторизация**
  - Вход по логину и паролю

- **Профиль пользователя**
  - Просмотр статистики по серверам
  - Пополнение кошелька ( пользователь вводит только цифры и ровно 16 цифр)
  - История покупок с возможностью скачивания чеков
  - Смена пароля
  - **Изменение никнейма (майнкрафт имени)**:
    - В профиле есть поле для ввода никнейма
    - Никнейм уникален — при попытке установить уже существующий ник выводится сообщение об ошибке
    - Можно изменить ник только если он свободен

- **Игровая карта и чат**
  - карта сервера
  - чат сервера
  - статистика игроков (топ игроков, ввод по нику и категория)

- **Услуги сайта**
  - Перечень донатов по сервера
  - Возможность пополнения кошелька

- **Правила**
  - Общие правила и правила для конкретных серверов
  - Правила по имени сервера — показываются при выборе
  - Внутри правила — название, описание, тема, якоря для быстрого перехода

- **Вики**
  - FAQ (часто задаваемые вопросы)
  - По категориям и вопросам — раскрытие текста
  - Возможность перейти к этапам вопроса

- **Техподдержка**
  - Кнопка обращения
  - После обращения — бот отправляет сообщение в Discord (логин, ID, тема, описание)
  - В техподдержке — команды `/sendhelp`, `/closedhelp` для работы с запросами

### 2. База данных (MySQL)
#### Таблицы:
| Название        | Описание                                                 |
|-----------------|----------------------------------------------------------|
| `users`       | Пользователи: логин, хеш пароля, роль (пользователь/поддержка), никнейм |
| `wallets`     | Баланс, история пополнений                              |
| `purchases`   | История покупок, дата, сумма, сервер, тип                |
| `receipts`    | Чеки: UUID, пользователь, что куплено, сумма, карта, дата  |
| `server`     | Информация о сервере: название, описание, донаты, правила  |
| `donations`   | Детали донатов и история покупок донатов               |
| `stats`       | Статистика по серверу, прохождениям, времени активности   |

- Никнейм хранится в таблице `users`, он должен быть уникальным — при регистрации или смене никнейма проверяются дубликаты
- Пароли — хранятся в виде хеша (например, bcrypt)
- Чеки сохраняются как файлы и в базе, доступны для скачивания

# Таблица `users`
Хранит информацию о зарегистрированных пользователях и их ролях.

| Поле             | Тип               | Ограничения                   | Описание                                    |
|------------------|-------------------|------------------------------|---------------------------------------------|
| `id`             | INT UNSIGNED     | PRIMARY KEY, AUTO_INCREMENT | Уникальный ID пользователя                  |
| `login`          | VARCHAR(50)      | UNIQUE, NOT NULL             | Логин для входа                            |
| `password_hash`  | VARCHAR(255)     | NOT NULL                     | Хэш пароля (например, bcrypt)              |
| `role`           | ENUM('user','support') | NOT NULL                 | Роль пользователя                           |
| `nickname`       | VARCHAR(30)      | UNIQUE, DEFAULT NULL         | Майнкрафт никнейм (может быть пустым)     |
| `created_at`     | TIMESTAMP        | DEFAULT CURRENT_TIMESTAMP    | Время регистрации                          |

Обратите внимание: Никнейм должен быть уникальным и проверяться при изменении или регистрации.

# Таблица `wallets`
Хранит баланс и историю пополнений каждого пользователя.

| Поле             | Тип               | Ограничения                   | Описание                                 |
|------------------|-------------------|------------------------------|------------------------------------------|
| `id`             | INT UNSIGNED     | PRIMARY KEY, AUTO_INCREMENT | Идентификатор записи                     |
| `user_id`        | INT UNSIGNED     | FOREIGN KEY к `users(id)`    | Владелец кошелька                        |
| `balance`        | DECIMAL(10,2)    | NOT NULL DEFAULT 0.00        | Текущий баланс                          |
| `last_updated`   | TIMESTAMP        | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | Последнее обновление баланса  |

---

# Таблица `purchases`
Записывает все покупки проходок и донатов.

| Поле             | Тип               | Ограничения                   | Описание                              |
|------------------|-------------------|------------------------------|---------------------------------------|
| `id`             | INT UNSIGNED     | PRIMARY KEY, AUTO_INCREMENT | ID покупки                            |
| `user_id`        | INT UNSIGNED     | FOREIGN KEY к `users(id)`    | Пользователь                        |

| `amount`         | DECIMAL(10,2)    | NOT NULL                     | Стоимость покупки                     |
| `date`           | DATETIME         | DEFAULT CURRENT_TIMESTAMP    | Время покупки                        |
| `details`        | TEXT             | NULL                         | Дополнительные сведения             |

| `issue_date`     | DATETIME         | DEFAULT CURRENT_TIMESTAMP    | Дата создания                            |

### Взаимосвязи между таблицами
- **`users`** связана с:
  - `wallets` — один к одному
  - `purchases` — один ко многим
  - `receipts` — один ко многим
  - `donations` — один ко многим
  - `stats` — один ко многим

- **`purchases`** связана с:
  - `users`
  - `servers`
  - `receipts`

- **`receipts`** связана с:
  - `users`
  - `purchases`

- **`donations`** связана с:
  - `users`
  - `servers`

- **`stats`** связана с:
  - `users`
  - `servers`

### 3. Серверная часть (на Go)
- Обработка API-запросов:
  - Регистрация, авторизация
  - Обновление профиля, смена пароля
  - Внесение изменений в никнейм — проверка на уникальность
  - Пополнение кошелька ( пользователь вводит только цифры и ровно 16 цифр)
  - Покупки проходки и донатов
  - Создание и сохранение чеков
  - Обработка запросов техподдержки и помощь

### 4. Дополнительные функции
- **Многоязычность**
  - Файлы `ru.json`, `en.json` для переводов
- **Темы сайта**
  - Переключение светлая/тёмная
- **Изменение профиля**
  - В профиле — возможность менять никнейм, при этом проверяется его уникальность
- **Подробнее о сервере**
  - Страница с описанием и кнопкой купить проход
- **Правила**
  - Общее право и для каждого сервера отдельно, через якоря
- **FAQ и Вики**
  - Структурированные по категориям, с раскрывающимися вопросами
- **Обратная связь / техподдержка**
  - В Discord — сообщения от бота
  - Команды `/sendhelp` и `/closedhelp`
