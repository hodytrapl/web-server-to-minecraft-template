# Техническое задание для учебного проекта: сайт по продаже проходок и донатов для Minecraft серверов

## Общая концепция
Разработка сайта и серверной части для управления покупками проходок и донатов:
- Регистрация и авторизация
- Хранение логина, хеша пароля, имени аккаунта
- Пополнение кошелька с условной обработкой
- Просмотр статистики по каждому серверу
- Чеки с возможностью скачивания
- История покупок
- Возможность менять никнейм (майнкрафт имя) в профиле
- Никнейм — уникальный, проверить его перед установкой
- Использование ролей (пользователь, техподдержка)

---

## Основные требования

### 1. Пользовательский интерфейс
- **Главная страница**
  - Описание проекта
  - Навигационная карта: главная, сервера, услуги, правила, вики, профиль, темы
  - Кнопка "Войти" и "Регистрация"
  - Переключение языка (ru/en) через файлы `ru.json`, `en.json`
  - Переключение темы (светлая/темная)
  
- **Форма регистрации**
  - Логин
  - Хешированный пароль
  - Имя аккаунта
  - Пополнение кошелька (с условиями обработки)

- **Авторизация**
  - Вход по логину и паролю

- **Профиль пользователя**
  - Просмотр статистики по серверам
  - Пополнение кошелька
  - История покупок с возможностью скачивания чеков
  - Смена пароля
  - **Изменение никнейма (майнкрафт имени)**:
    - В профиле есть поле для ввода никнейма
    - Никнейм уникален — при попытке установить уже существующий ник выводится сообщение об ошибке
    - Можно изменить ник только если он свободен

- **Каталог серверов**
  - список серверов
  - при нажатии — страница с информацией:
    - имя сервера
    - описание
    - иконка
    - список донатов
    - возможность покупки проходки

- **Услуги сайта**
  - Перечень донатов по серверам
  - Возможность пополнения кошелька

- **Правила**
  - Общие правила и правила для конкретных серверов
  - Правила по имени сервера — показываются при выборе
  - Внутри правила — название, описание, тема, якоря для быстрого перехода

- **Вики**
  - FAQ (часто задаваемые вопросы)
  - По категориям и вопросам — раскрытие текста
  - Возможность перейти к этапам вопроса

- **Техподдержка**
  - Кнопка обращения
  - После обращения — бот отправляет сообщение в Discord (логин, ID, тема, описание)
  - В техподдержке — команды `/sendhelp`, `/closedhelp` для работы с запросами

---

### 2. База данных (MySQL)
#### Таблицы:
| Название        | Описание                                                 |
|-----------------|----------------------------------------------------------|
| `users`       | Пользователи: логин, хеш пароля, роль (пользователь/поддержка), никнейм |
| `wallets`     | Баланс, история пополнений                              |
| `purchases`   | История покупок, дата, сумма, сервер, тип                |
| `receipts`    | Чеки: UUID, пользователь, что куплено, сумма, карта, дата  |
| `servers`     | Информация о серверах: название, описание, донаты, правила  |
| `donations`   | Детали донатов и история покупок донатов               |
| `stats`       | Статистика по серверу, прохождениям, времени активности   |

- Никнейм хранится в таблице `users`, он должен быть уникальным — при регистрации или смене никнейма проверяются дубликаты
- Пароли — хранятся в виде хеша (например, bcrypt)
- Чеки сохраняются как файлы и в базе, доступны для скачивания

---

### 3. Серверная часть (на Go)
- Обработка API-запросов:
  - Регистрация, авторизация
  - Обновление профиля, смена пароля
  - Внесение изменений в никнейм — проверка на уникальность
  - Пополнение кошелька
  - Покупки проходок и донатов
  - Создание и сохранение чеков
  - Обработка запросов техподдержки и помощь

### 4. Дополнительные функции
- **Многоязычность**
  - Файлы `ru.json`, `en.json` для переводов
- **Темы сайта**
  - Переключение светлая/тёмная
- **Изменение профиля**
  - В профиле — возможность менять никнейм, при этом проверяется его уникальность
- **Подробнее о сервере**
  - Страница с описанием и кнопкой купить проход
- **Правила**
  - Общее право и для каждого сервера отдельно, через якоря
- **FAQ и Вики**
  - Структурированные по категориям, с раскрывающимися вопросами
- **Обратная связь / техподдержка**
  - В Discord — сообщения от бота
  - Команды `/sendhelp` и `/closedhelp`
